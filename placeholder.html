번역 검수 플레이스홀더 도구

엑셀/CSV 플레이스홀더 제거기 (HTML 단일 파일, 배치 지원)

이 도구는 브라우저 안에서만 동작합니다. 파일이 서버로 전송되지 않습니다.
삭제는 엑셀의 “찾기/바꾸기에서 바꿀 내용 공란”과 동일하게, 해당 문자열만 정확히 제거합니다(대소문자 구분, 모든 위치 전부 제거).
드래그 앤 드롭으로 파일을 넣거나, 아래 버튼으로 선택하세요.
여러 파일을 한 번에 처리하는 경우, 결과는 ZIP 1개로 다운로드됩니다(브라우저 다중 다운로드 차단을 피하기 위한 방식).
분석 후 각 파일의 최소 정보(파일명, 시트 수, 행 수, 텍스트 글자 수, 텍스트 셀 수)를 표시합니다.
제약과 불안정 요소(반드시 확인)
추가가 사실상 불가능하거나 지원하지 않는 것
- 암호 또는 암호화된 엑셀 파일(.xlsx) 처리
- .xls(구형 바이너리) 및 매크로(.xlsm) 완전 보존 보장

양(파일과 로그 크기)에 따라 불안정해질 수 있는 것
- 매우 큰 XLSX(시트, 셀, 텍스트가 많음)에서는 분석과 변환 자체가 느려지거나 브라우저 탭이 멈출 수 있습니다
- 리포트 “전체(원문 포함)”은 리포트 파일이 커져서 느려질 수 있습니다

원본 보존
- XLSX는 다시 생성하기보다 zip 내부에서 시트 XML과 sharedStrings 텍스트 노드만 최소 수정하는 방식이라 보존성이 높은 편입니다.
- 다만 엑셀의 모든 고급 요소를 항상 100% 보장한다고 단정할 수는 없습니다.
둘 중 하나만 선택해서 사용합니다.
줄바꿈이 포함된 문자열은 \n 으로 입력하실 수 있습니다.
입력한 문자열을 그대로(리터럴) 제거합니다. 공백도 문자열의 일부로 취급합니다.
왼쪽(찾을 문자열)과 오른쪽(바꿀 문자열)을 입력하면 모든 파일에 일괄 적용됩니다. 줄바꿈은 \n 으로 입력하실 수 있습니다.
규칙은 위에서 아래 순서대로 적용됩니다.

적용 범위(XLSX)

“파일별 시트 선택”은 분석 후 각 파일에서 별도로 선택합니다.
